# 基于特色创新项目的岭南红橙病虫害智能监测与校本教研一体化平台技术规范文档

> **近期功能修订说明（2026-05-30）**：系统已根据当前实现补充数据采集、数据标注、模型训练、训练结果可视化、农户档案 CSV 导入和模型权重发布规范。数据采集流程已调整为“仅使用搜索关键词采集”，不再强制选择目标类别、参照图或图片尺寸阈值；采集后的图片统一进入未分类原始样本目录，由数据标注阶段再绑定 12 类病虫害类别。模型训练结束后自动发布 `best.pt` 到项目 `models/` 目录，并在训练结果页自动解析曲线、混淆矩阵和预测示例图。

## 一、 项目总则

### 1.1 建设背景与意义
岭南红橙是广东省廉江市特色优势农产品和国家地理标志产品，具有果形端正、果肉橙红、汁多化渣、甜中带酸、清甜带香等鲜明品质特征，是廉江“中国红橙之乡”区域品牌的重要载体。近年来，廉江围绕红橙产业链推进区域公用品牌、产品标准、质量追溯和绿色防控体系建设，病虫害精准识别与规范化防治已成为保障鲜果品质、稳定产量和维护品牌信誉的关键环节。

在实际种植管理中，岭南红橙果园仍面临黄龙病、溃疡病、红蜘蛛、木虱、潜叶蛾、蚜虫等多类柑橘病虫害威胁。病害早期症状与缺素、日灼、机械伤、药害等现象容易混淆；红蜘蛛、木虱若虫等小目标虫害体型微小，田间人工巡查容易漏判。若防治窗口把握不准，可能造成果面商品性下降、产量损失、农药使用不规范和采前安全间隔期管理风险。

本项目依托岭南红橙现代农业产业园、红橙专业合作社及基层农技服务场景，研发一套**全本地化运行、零网络依赖、轻量级部署**的**基于特色创新项目的岭南红橙病虫害智能监测与校本教研一体化平台**。系统通过轻量化目标检测模型、田间图像数据集、结构化绿色防治知识库和本地台账模块，实现“无需联网、一键检测、精准开方、本地留痕”，服务岭南红橙标准化生产、绿色防控和质量追溯。

### 1.2 核心建设目标
1.  **零网络依赖**：系统各项功能（图像读取、模型推理、结果标注、方案匹配、报表导出）实现100%全本地化离线运行。
2.  **高精度识别**：针对岭南红橙高发的12类典型病虫害，其模型检测平均精度（mAP@0.5）不低于91%，尤其针对红蜘蛛、木虱等微型小目标具备极高敏感度。
3.  **秒级响应**：单张图片推理及综合防治方案匹配总耗时不超过1.2秒。
4.  **极低硬件门槛**：无需配置昂贵的工业级GPU服务器或显卡，在普通家用或办公电脑（如Intel i3、8G内存）上即可流畅运行。
5.  **适老化与精细化防治**：提供全图形化、一键式的极简交互界面，自动开具“物理防治、生物防治、低毒化学防治”三位一体的绿色处方，降低化学农药依赖。
6.  **品质与溯源支撑**：围绕岭南红橙鲜果商品性、采收期安全间隔期、果园区块台账和区域品牌追溯要求，形成可查询、可导出、可审计的本地检测记录。

### 1.3 适用对象
* **岭南红橙种植农户**：进行日常田间巡检与快速病虫害自查。
* **基层种植合作社**：指导社员科学用药，统一防治标准，建立种植台账。
* **产业园种植管理单位/农业技术推广部门**：用于农业技术下乡培训、应急病虫害普查及区域内病虫害数据的收集与分析。
* **收购分级与品牌运营主体**：辅助记录果园病虫害风险、采前用药合规性与商品果质量追溯信息。

### 1.4 岭南红橙专项业务约束
* **品类定位**：系统面向岭南红橙鲜果生产和果园管理，不采用陈皮加工或药材生产场景的判定逻辑。
* **识别部位**：数据集需覆盖叶片、嫩梢、枝干、花蕾、幼果、膨大果、成熟果及叶背虫害等关键部位。
* **采集季节**：样本应覆盖春梢、夏梢、秋梢、开花坐果、果实膨大、转色成熟和采收前后等阶段。
* **质量关注点**：除病虫害本身外，应重点关注果面斑点、虫伤、煤烟病污染、异常黄化、落果风险等影响鲜果商品等级的视觉特征。
* **绿色防控要求**：推荐处方必须优先给出农业防治、物理诱控、生物防治与生态调控措施，化学药剂建议需绑定安全间隔期、适用物候期和采收风险提示。

## 二、 系统总体技术架构

系统采用完全解耦的“算法推理层、应用交互层、本地数据层”三层架构设计，确保系统在无任何外部网络及云端服务交互的情况下，依然能够高效、稳定地提供全链条智能服务。

```
+-----------------------------------------------------------------------+
|                         应用交互层 (极简桌面客户端)                   |
| 模块：图像/视频导入 | 摄像头调用 | 结果可视化标注 | 方案一键生成 | 台账管理 |
+-----------------------------------------------------------------------+
                                   │
                     调用/传输 (本地 ONNX Runtime 接口)
                                   ▼
+-----------------------------------------------------------------------+
|                         算法推理层 (YOLOv8 边缘推理)                  |
| 模块：图像预处理 | 目标检测 (YOLOv8s-XH) | 分类计数 | 严重程度分级计算 |
+-----------------------------------------------------------------------+
                                   │
                           检索/匹配 (本地嵌入式存储)
                                   ▼
+-----------------------------------------------------------------------+
|                         本地数据层 (结构化知识库与台账)                |
| 数据库：12类病虫害物候期防治处方 | 本地检测历史日志 | 农户/果园分区档案   |
+-----------------------------------------------------------------------+
```

### 2.1 算法推理层
算法推理层是本系统的核心。系统摒弃传统的云端服务器算力支撑，将经过专项优化和量化剪枝后的 YOLOv8s 模型（暂定代号：`YOLOv8s-XH`）封装于本地客户端内。
* **输入流**：支持本地JPEG/PNG/BMP等主流格式图像导入，同时支持标准UVC协议的便携式数字显微镜/摄像头实时视频流输入。
* **核心功能**：执行高动态图像预处理、多尺度特征提取、病灶边界框回归、目标分类归纳以及群落计数。针对微型虫害（如红蜘蛛、蚜虫等），算法层在特征金字塔网络（FPN）中引入小目标检测层，以提升弱小特征的捕捉能力。

### 2.2 应用交互层
应用交互层负责与用户进行直接交互。基于跨平台图形库开发，适配Windows 10/11及信创国产Linux系统（如统信UOS、麒麟软件）。
* **核心理念**：极简适老化。界面避免多级菜单和复杂的参数配置，主功能区域采用“大字体、高对比度、单键触发”模式。
* **核心流程**：农户只需拖入照片或点击“拍照”按钮，系统立即在界面直观标出病灶位置，并同步展示匹配的专家处方，支持防治方案的一键打印或导出。
* **界面技术栈**：桌面端采用 PyQt5 + PyQt-Fluent-Widgets（`qfluentwidgets`）构建，核心按钮、输入框、卡片、导航和提示组件优先使用 `PrimaryPushButton`、`TransparentPushButton`、`LineEdit`、`CardWidget/SimpleCardWidget`、`InfoBar` 等 Fluent Design 风格组件。
* **训练闭环入口**：应用交互层集成数据采集、内置标注、训练数据提交、YOLOv8 本地训练、训练结果可视化展示和权重自动发布，支持从原始图片到可部署模型的本地闭环。

### 2.3 本地数据层
本地数据层负责本地数据的持久化存储与专家规则检索，采用轻量级、免配置的本地嵌入式数据库（如SQLite）进行数据管理。
* **知识库检索**：根据算法层输出的病虫害类别标签、置信度以及用户选择的果树物候期（如萌芽期、幼果期、挂果期、采收期），检索对应的动态防治方案。
* **数据安全**：所有农户资料、检测地点、发病历史、用药记录均100%保存在本地硬盘中，不向任何外部公有云上传数据，彻底消除种植隐私与数据外泄的风险。

## 四、 核心功能模块详细规范

### 4.1 智能检测分类模块

#### 4.1.1 12类岭南红橙常见病虫害识别矩阵
系统必须具备精准区分以下12类廉江本地高发、高危害病虫害的能力，并能适应不同光照、背景干扰及拍摄角度。

1.  **柑橘黄龙病（Citrus Huanglongbing / HLB）**：精准识别斑驳状黄化叶片、青果、红鼻子果等关键特征，触发高级别红色预警。
2.  **柑橘溃疡病（Citrus Canker）**：识别叶片、枝条及果实上隆起的火山口状褐色木栓化病斑。
3.  **柑橘疮痂病（Citrus Scab）**：识别叶片及幼果上突起的黄褐色漏斗状或尖锥状疮痂。
4.  **柑橘炭疽病（Citrus Anthracnose）**：识别叶尖或叶缘出现的枯死斑，以及果实上的下陷褐色圆斑。
5.  **柑橘黑斑病（Citrus Black Spot）**：识别果实表面红褐色、中央凹陷并带有黑色边缘的典型黑斑。
6.  **柑橘红蜘蛛（Citrus Red Mite）**：对叶片及果实表面密集的红色微型虫体进行小目标定位与密集计数，评估失绿白点严重度。
7.  **柑橘木虱（Citrus Psyllid）**：精准识别黄龙病唯一昆虫媒介，包含成虫（尾部上翘45度特征）及白色/黄色若虫群落。
8.  **柑橘潜叶蛾（Citrus Leaf Miner）**：识别幼虫在嫩叶内潜食后留下的银白色蜿蜒隧道及叶片卷曲特征。
9.  **柑橘蚜虫（Citrus Aphids）**：识别聚集在嫩梢、幼叶背面的黑色或绿色虫体，自动计算群落覆盖区域。
10. **柑橘介壳虫（Citrus Scale Insects）**：识别吹绵介壳虫、红圆介壳虫等固定在枝干、叶片上的盾形或棉絮状虫体。
11. **柑橘花蕾蛆（Citrus Flower Bud Midge）**：识别危害导致的花蕾异常膨大（“灯笼状”花蕾）及内部幼虫受损特征。
12. **柑橘黑刺粉虱（Citrus Blackfly）**：识别叶背密集的黑色若虫及边缘白色蜡质圈，以及伴生的煤烟病特征。

#### 4.1.2 岭南红橙田间图像专项识别要求
* **叶片类样本**：需覆盖正面、背面、嫩叶、老叶、缺素相似症状、药害相似症状和自然黄化干扰样本，降低黄龙病、缺素和日灼误判。
* **果实类样本**：需覆盖幼果、膨大果、转色果、成熟果，重点识别溃疡病、黑斑病、疮痂病、虫伤、煤烟污染等影响果面商品性的病虫害表现。
* **虫害小目标样本**：红蜘蛛、木虱、蚜虫、黑刺粉虱等需补充 USB 显微镜/手机近拍数据，保证叶背、嫩梢、虫卵、若虫和成虫均有样本。
* **复杂场景样本**：需纳入强光、阴影、雨后、泥土背景、枝叶遮挡、果袋边缘、人工拿叶/拿果等真实田间干扰场景。
* **负样本要求**：数据集中必须包含健康叶、健康果、机械伤、裂果、日灼、缺素、鸟啄和采后擦伤等非病虫害样本，用于降低误报。

#### 4.1.3 结果可视化标注规范
* **病灶定界框（Bounding Box）**：对识别到的每一个病斑或虫体，系统需使用半透明彩色矩形框进行精准覆盖。不同种类的病虫害需使用不同颜色代码进行区分（例如：黄龙病使用闪烁红框、红蜘蛛使用亮黄框）。
* **标签文本（Label）**：每个定界框顶部需清晰标注`[中文病虫害名称] [置信度分数]`，如：`柑橘溃疡病 94.5%`。
* **严重程度自动分级**：系统需根据图像中病斑面积占比或虫体绝对计数，自动计算严重程度分级。分级指标分为三级：
    * **轻度（Green）**：单张叶片/果实病斑面积 $<5\%$，或虫体计数 $<5$ 头。
    * **中度（Amber）**：病斑面积占比在 $5\% - 20\%$ 之间，或虫体计数在 $5 - 20$ 头之间。
    * **重度（Red）**：病斑面积占比 $>20\%$，或虫体计数 $>20$ 头，或确认为黄龙病。
* **密集自动计数**：针对红蜘蛛、蚜虫等群聚性虫害，系统需具备“点计数”能力，在界面右侧面板自动输出当前图片中虫害的总头数，避免农户人工清点。

### 4.2 精准防治方案推送模块

#### 4.2.1 动态联动机制
系统检测完成后，界面下方立即联动展现该特定病虫害的结构化防治方案。方案展示必须结合“果树物候期”进行动态二次筛选。系统提供快捷下拉菜单供用户选择当前果树状态：
1.  **春梢期/萌芽期**
2.  **开花期/谢花期**
3.  **生理落果期/幼果期**
4.  **果实膨大期/挂果期**
5.  **采收期/冬剪期**

#### 4.2.2 三位一体绿色处方规范
针对每一类病虫害的特定阶段，推送的处方必须严格划分并展示以下三类，全面践行绿色农业、控药减施的指导思想：
* **处方A：物理防治（优先）**：包含使用黄板诱杀（针对木虱、蚜虫）、频振式杀虫灯、人工剪除病枝烧毁、大棚防虫网隔离等非药物手段。
* **处方B：生物防治（安全）**：包含释放捕食螨（针对红蜘蛛）、喷施苏云金杆菌（BT，针对潜叶蛾）、利用寄生蜂（针对介壳虫）等生态调控手段。
* **处方C：科学化学防治（合规）**：
    * **推荐药剂**：仅允许推荐符合国家绿色食品生产标准、低毒低残留的登记药剂（如：螺螨酯、阿维菌素、高效氯氟氰菊酯等）。严格禁止出现任何国家明令禁止的高毒、高残留农药。
    * **规范用量**：明确写明稀释倍数（如：`2000-3000倍液`）或每亩标准用量，严禁使用“适量”、“大剂量”等模糊表述。
    * **安全间隔期（PHI）**：**必须突出显示**该药剂喷施后距离岭南红橙采收的安全天数（如：`安全间隔期：21天`），从源头上规避鲜果农药残留超标风险。
    * **注意事项**：写明药剂混用禁忌（如：`请勿与碱性农药混用`）及天气喷施建议。
    * **采前风险提示**：当用户选择“成熟转色期/采收期”且药剂 PHI 超过预计采收剩余天数时，系统必须给出黄色或红色警示，并优先推荐非化学防治或采后清园措施。

### 4.3 本地数据管理模块

#### 4.3.1 日志自动化存贮
每次检测完成，系统在经农户确认后（或设置自动保存），将以下要素以结构化日志形式写入本地 SQLite 数据库：
* `ID`：系统全局唯一递增序列号。
* `DateTime`：精确到秒的检测时间戳（格式：`YYYY-MM-DD HH:MM:SS`）。
* `Sample_Source`：图像来源类型（“本地导入” 或 “实时拍照”）。
* `Target_Disease`：识别出的病虫害主类别。
* `Confidence`：模型最高置信度数值。
* `Severity_Level`：轻度/中度/重度分级。
* `Count_Value`：虫害定量计数结果（非虫害类则记录为0）。
* `Farmer_Name` 与 `Orchard_Block`：用户自定义输入的农户姓名与果园区块编号（支持留空，便于归档）。
* `Prescription_Snapshot`：当时推送的化学处方核心药剂快照。
* `Phenophase`：红橙物候期（春梢、开花、幼果、膨大、转色成熟、采后清园等）。
* `Harvest_Risk`：采前用药风险等级（安全/需关注/禁止推荐）。
* `Trace_Batch`：可选填写的果园批次或合作社收购批次，用于与品牌溯源台账关联。

#### 4.3.2 报表一键导出规范
系统提供“历史台账”管理面板，支持用户按时间段（如最近一周、最近一月）、按农户姓名或按病虫害类别进行本地组合筛选。
* **一键导出**：点击“导出Excel报表”按钮，系统需在3秒内于用户指定盘符下生成一份标准 `.xlsx` 格式的报表文件。
* **报表用途**：该文件可直接提供给农业产业园管理单位或种植合作社作为标准化种植管理台账、农药施用合规性审计以及病虫害发生规律分析的原始数据支撑。

#### 4.3.3 农户档案 CSV 批量导入规范
农户/果园档案模块除支持界面逐条新增、修改、删除外，还需支持 CSV 批量导入和模板生成，便于合作社或产业园一次性维护基础档案。
* **CSV 模板**：系统提供 `docs/农户档案导入模板.csv`，界面中提供“下载 CSV 模板”按钮，可生成 Excel 兼容的 UTF-8 BOM CSV 文件。
* **必填字段**：`农户姓名`、`果园区块`、`联系电话`、`果园位置`。`备注`为可选字段。
* **字段兼容**：导入器兼容中文表头和常见英文表头（如 `farmer_name`、`orchard_block`、`phone`、`location`、`notes`）。
* **导入规则**：以“农户姓名 + 果园区块”为唯一组合键；重复导入时自动更新电话、位置和备注，不生成重复档案。
* **异常反馈**：CSV 缺少必填列时直接阻止导入；单行缺少必填值时跳过该行并在界面展示错误行号和缺失字段。

### 4.4 数据采集与标注数据集管理模块

#### 4.4.1 公开图片采集规范
数据采集模块用于快速扩充原始样本池，适用于研究、测试、原型验证和辅助样本整理；正式训练仍应优先使用自采图片、授权数据或许可证明确允许的数据集。
* **采集输入**：仅使用搜索关键词作为采集条件，例如 `岭南红橙 黄龙病 叶片`。系统不再要求选择目标类别、参照图或最低图片尺寸。
* **搜索引擎**：采集入口不限于 Bing，支持 `Bing`、`Google`、`百度` 以及 `Bing + Google + 百度` 混合模式；混合模式下系统合并候选图片 URL 并自动去重。
* **类别绑定时机**：采集阶段不绑定 12 类病虫害类别，所有类别归属由数据标注阶段通过人工框选和类别选择完成，避免搜索结果先验类别污染训练标签。
* **保存目录**：采集成功的图片按关键词保存至 `runtime_data/lianjiang_hongcheng_dataset/raw/search_关键词/`，并统一转存为 `.jpg`。
* **去重规则**：采集时执行 URL 去重和 SHA256 内容哈希去重；重复图片不写入 raw 目录。
* **采集清单**：系统维护 `collection_manifest.csv`，记录本地路径、关键词、来源 URL、宽高、SHA256、采集时间等信息；未分类采集记录的类别字段标记为“未分类”。
* **停止与重启**：采集线程支持用户主动停止；停止后重新开始采集时需重新初始化任务状态，避免残留线程或旧停止标记影响下一轮采集。

#### 4.4.2 内置标注工具规范
系统内置 LabelImg 风格的矩形框标注能力，标注阶段负责真正确定病虫害类别。
* **图片管理**：支持导入单张/多张图片、导入目录、删除当前导入图片、清空全部导入图片。
* **框选标注**：用户在图片上拖拽生成 YOLO 矩形框，选择 12 类病虫害之一作为框类别。
* **标注列表**：界面展示当前图片的所有标注框，支持删除选定标注框、清空当前图片标注框。
* **已标数据管理**：系统展示已标注数据列表，支持单选或多选已标数据集记录并批量删除，同时删除对应 raw 原图、标签文件和训练/验证/测试拆分副本。
* **提交约束**：数据标注与模型训练必须拆开；只有标注数据集提交成功后，模型训练入口才允许启动。提交时系统校验 raw 图片数量、已标注图片数量、训练集/验证集有效图片数量和 `data.yaml` 完整性。

### 4.5 模型训练与训练结果可视化模块

#### 4.5.1 训练数据选择与训练执行规范
模型训练模块支持使用项目已提交数据集，也支持选择本地 YOLO 数据集 `data.yaml`。
* **项目数据集**：默认读取 `runtime_data/lianjiang_hongcheng_dataset/data.yaml`，由数据标注页提交生成。
* **本地数据集**：用户可通过“选择本地数据集”选择外部 YOLO 数据集配置文件；系统必须校验 `train` 与 `val` 路径存在且包含图片。
* **训练入口**：训练通过 `python -m xhgan.train_yolo` 启动，避免直接执行 `ultralytics` 包导致 `No module named ultralytics.__main__` 错误。
* **训练运行目录**：训练输出保存至 `runtime_data/training_runs/yolov8s_xh*/`，其中 `weights/best.pt` 为验证集指标最优权重。
* **模型发布**：训练正常结束后，后台自动将训练目录中的 `weights/best.pt` 复制到项目 `models/best.pt`，并同步复制为 `models/yolov8s_xh_best.pt` 供系统自动加载；若存在 `last.pt`，同步保存为 `models/yolov8s_xh_last.pt`。

#### 4.5.2 训练结果可视化规范
训练结果页用于自动展示并解释 YOLOv8 训练产物。
* **训练曲线**：自动解析 `results.csv`，生成并展示训练集/验证集损失变化曲线。
* **Precision / Recall 曲线**：展示精度与召回率随 epoch 的变化，并对误检、漏检倾向给出文字解析。
* **mAP 指标曲线**：展示 `mAP50` 与 `mAP50-95` 指标曲线，辅助判断模型收敛质量和泛化能力。
* **混淆矩阵**：自动加载 YOLOv8 生成的混淆矩阵图，分析类别间混淆风险。
* **预测示例图**：自动加载 `val_batch*_pred.jpg` 等预测结果图，作为训练后直观质检样例。
* **总结输出**：在所有图表之后自动生成训练结果总结，提示是否存在欠拟合、过拟合、精度偏低、召回偏低或结果不完整等风险。
* **稳定性要求**：刷新按钮不得将 PyQt `clicked(bool)` 参数误传为训练目录路径；`refresh_results()` 必须兼容空路径、字符串路径、`Path` 对象和按钮布尔参数，避免出现 `TypeError: expected str, bytes or os.PathLike object, not bool`。

## 五、 项目实施周期与进度规划

本项目严格控制总工期为 **45天**。项目实行“里程碑管理制”，各阶段无缝衔接，以确保系统按时按质交付上线。

### 5.1 阶段里程碑详细划分

```
[第1-10天]   G1: 数据采集与标注  ======► 产出: 10,000+样本专属标注数据集
[第11-25天]  G2: 模型训练与优化  ======► 产出: 最优 YOLOv8s-XH 模型及测试报告
[第26-38天]  G3: 客户端与知识库集成 ==► 产出: 离线集成客户端软件安装包
[第39-45天]  G4: 测试优化与交付  ======► 产出: 农户操作手册、项目整体验收
```

#### 阶段一：数据筑基——样本采集与全要素标注（第 1 - 10 天）
* **主要任务**：
    1. 联合农业技术顾问，深入岭南红橙核心种植基地、合作社示范园和不同树龄果园，实地拍摄黄龙病、溃疡病、红蜘蛛、木虱等12类病虫害在不同光照条件、不同发病时期的原始照片。
    2. 按“叶片、嫩梢、枝干、花蕾、幼果、成熟果、叶背虫害、健康负样本”建立采集清单，确保数据覆盖鲜果生产全流程。
    3. 使用系统数据采集模块按关键词收集公开图片辅助样本，采集图片先进入 `raw/search_关键词/` 未分类目录，后续由人工标注阶段绑定类别。
    4. 开展图像数据清洗，剔除严重模糊、过度曝光、主体不可辨、类别不确定的无效样本；对疑似黄龙病和木虱样本需经农技顾问复核。
    5. 使用内置标注模块 / LabelImg / Labelme 工具进行全要素矩形框和类别标注，完成数据集扩增（通过翻转、裁剪、色彩均衡、阴影模拟、雨后场景增强等手段）。
* **阶段产出与考核指标**：构建包含不少于 10,000 张高质量图像的岭南红橙专属病虫害标准数据集，其中果实类样本、叶片类样本、小目标虫害样本、健康负样本均需满足最低占比要求；标注框准确率达 100%。

#### 阶段二：模型攻坚——算法训练与超参调优（第 11 - 25 天）
* **主要任务**：
    1. 基于 YOLOv8s 轻量化基础架构搭建本地训练环境，载入预训练权重。
    2. 针对红蜘蛛、木虱等微型/小目标，定制化调整网络锚框（Anchors）大小，引入空洞卷积或注意力机制（如CBAM）以强化局部特征提取能力。
    3. 执行多轮（不低于 300 Epochs）迭代训练，进行超参数（学习率、动量、权重衰减等）调优。
    4. 训练完成后自动将 `weights/best.pt` 发布至项目 `models/best.pt` 和 `models/yolov8s_xh_best.pt`。
    5. 将训练完毕的最优权重文件导出为跨平台、高性能的 `.onnx` 格式，并利用 ONNX Runtime 进行本地 CPU 推理性能量化测试。
    6. 通过训练结果可视化页生成 Loss、Precision/Recall、mAP、混淆矩阵和预测示例图解析。
* **阶段产出与考核指标**：输出最优解算模型文件 `models/best.pt`、`models/yolov8s_xh_best.pt`、`yolov8s_xh_best.onnx`，提交《模型训练与综合性能测试报告》，确保测试集 mAP@0.5 $\ge 91\%$。

#### 阶段三：系统集成——离线客户端研发与知识库结构化（第 26 - 38 天）
* **主要任务**：
    1. 编写轻量化本地 GUI 桌面客户端代码，实现图像载入、视频捕获、模型热加载、检测框本地渲染等核心业务逻辑。
    2. 将农业技术顾问编写并审核的岭南红橙 12 类病虫害绿色防治标准方案进行结构化转换，导入本地嵌入式 SQLite 数据库，建立“病虫害类别 + 物候期 + 采前风险等级”复合索引机制。
    3. 开发本地日志模块与 Excel 导出引擎（基于轻量化文件写入组件，无需本地电脑安装 MS Office 软件即可运行）。
    4. 增加果园区块、采收批次、合作社名称、预计采收日期等字段，为岭南红橙质量追溯和绿色生产台账提供接口。
* **阶段产出与考核指标**：完成系统全链路集成，输出首个离线免安装桌面客户端程序（Alpha/Beta版），实现“无网状态下”一键全流程通畅运行。

#### 阶段四：田间测试——现场Debug与规范化交付（第 39 - 45 天）
* **主要任务**：
    1. 将系统部署到普通家用笔记本电脑中，携带便携式 USB 显微镜摄像头，深入偏远果园进行“断网、无网”条件下的田间实地功能测试。
    2. 收集农户、合作社管理人员、收购分级人员和农技员的现场使用反馈，针对小目标漏检、果面病斑误判、界面字体大小、操作流程舒适度进行最后一轮 Bug 修复与体验优化。
    3. 对训练结果刷新、采集停止后重启、删除标注数据、CSV 导入异常提示等关键交互进行回归测试。
    4. 编写并精简适老化配套图文材料，整理全部交付物。
* **阶段产出与考核指标**：系统功能稳定、无致命闪退错误；完成项目全部交付清单，通过产业园种植管理单位的最终现场验收。

## 摘要

本报告依据《物体检测项目流程》所定义的**九阶段标准流程**，结合**岭南红橙现代农业产业园**核心种植区病虫害精准防治需求，对"基于特色创新项目的岭南红橙病虫害智能监测与校本教研一体化平台"进行逐阶段可行性分析，并辅以技术、经济、操作、法律、进度五维综合评估。研究结论：项目在每个阶段均具备成熟的技术路径与可控的风险，**综合可行性评分 4.8 / 5，建议立项实施**。

**关键词**：岭南红橙、岭南红橙、病虫害检测、YOLOv8、ONNX Runtime、本地离线、九阶段流程、可行性研究

本报告论证项目"要不要做、能不能做"，与《技术规范文档》"怎么做"互为补充。论证岭南红橙病虫害本地智能检测防治系统在 **45 天工期、4 人团队**约束下，满足产业园种植管理需求的可行性。

### 1.2 KPI 可达性论证

| KPI | 设定 | 业界基线 / 实测预估 | 可达性 |
| :--- | :--- | :--- | :--- |
| 12 类综合 mAP@0.5 ≥ 91% | 自有 10k 样本充分训练 | 国内同类柑橘病害论文报告 88~93% | ✓ 边际可达 |
| 红蜘蛛 / 木虱 Recall ≥ 85% | P2 浅层 head + CBAM 注意力 | 加 P2 后业界小目标 Recall 提升 5~8 pt | ✓ |
| HLB 漏检率 ≤ 4% | 阳性样本充分扩增 + 误检偏向 | YOLOv8 类别敏感策略可控 | ✓ |
| CPU 单图 ≤ 0.8 s | YOLOv8s INT8 在 i3-10100 实测 ~600 ms | ✓ 富余 |
| 综合响应 ≤ 1.2 s | 推理 600 ms + IO 100 ms + SQLite 50 ms | ✓ 富余 |
| 摄像头 ≥ 15 fps | 640×480 流 + INT8 推理 | ✓ |
| 安装包 ≤ 150 MB | ONNX 7 MB + Qt 80 MB + 知识库 5 MB | ✓ |
| 完全离线 | ONNX Runtime 本地 + SQLite 嵌入式 | ✓ 架构保障 |

**风险点**：mAP ≥ 91% 是边际目标，依赖数据集质量。**缓解**：阶段一标注准确率 100% 验收 + 农业顾问全程审核。

**结论**：需求阶段可行性 ★★★★★。

## 4 阶段四：模型选择 —— 可行性

### 4.1 选型论证

| 候选 | 速度 | 精度 | CPU 部署 | 小目标 | 选用 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| Faster R-CNN | 慢 | 高 | 难 | 一般 | × |
| SSD | 中 | 中 | 一般 | 差 | × |
| YOLOv5s | 快 | 较高 | 友好 | 一般 | × |
| **YOLOv8s（定制 XH）** | 快 | 高 | 友好 | **优**（加 P2 head） | **✓** |
| YOLOv9 / v10 | 快 | 更高 | 友好 | 优 | 备选（生态较新） |
| RT-DETR | 中 | 高 | 中等 | 优 | 备选（CPU 偏慢） |

### 4.2 选 YOLOv8s-XH 的关键依据

1. **生态最佳**：GitHub Star ≥ 30k，月度更新，中文资源丰富，便于廉江本地维护接管
2. **CPU 部署友好**：ONNX 一键导出 + ORT INT8 量化，在 i3 级 CPU 上仍可亚秒级
3. **小目标能力**：Anchor-Free + P2 head 改造在红蜘蛛 / 木虱场景表现优秀
4. **统一 API**：训练 → 评估 → 导出 一行调用，节省 45 天工期内的调试成本
5. **多任务可扩展**：未来扩展到病斑分割、严重程度回归无需换栈

**结论**：★★★★★。

## 7 阶段七：模型优化 —— 可行性

### 7.1 优化路径可行性

| 优化手段 | 可行性 | 是否启用 |
| :--- | :--- | :--- |
| ONNX 导出 | ✓ `yolo export format=onnx` 官方一行命令 | **启用** |
| INT8 静态量化 | ✓ ORT PTQ，校准集 500 张即可 | **启用** |
| ORT 算子融合 / 多线程 | ✓ Session Options 配置即可 | **启用** |
| FP16 | △ CPU 收益小 | 备选 |
| 剪枝 / 蒸馏 | △ 工期不允许 | 路线图 V2 |

### 7.2 时延达标论证

i3-10100 / 8 GB 基准机预估（基于业界 YOLOv8s INT8 ONNX 实测）：

| 形态 | CPU 单图 | 综合响应 |
| :--- | ---: | ---: |
| FP32 ONNX | ~900 ms | ~1300 ms（超标） |
| **INT8 ONNX** | **~600 ms** | **~1000 ms（达标）** |

**结论**：INT8 形态下 KPI-04 / KPI-05 均富余满足。

### 7.3 风险

- **风险**：INT8 量化可能导致 mAP 下降 1~3 点
  **缓解**：校准集覆盖 12 类 × 5 物候期；量化后回归测试；mAP 损失 > 1.5 pt 时回退至 FP32 形态（牺牲速度）
- **风险**：低端 CPU 不支持 AVX2 指令集导致 ORT 性能急剧下降
  **缓解**：硬件要求明确写入 NFR（Intel 第四代及以上）

**结论**：★★★★☆。

## 12 结论与建议

### 12.1 总结论

按《物体检测项目流程》九阶段逐项分析，每阶段技术成熟、工具完备、风险可控；五维综合评估得分 **4.8 / 5**。本项目在 **45 天工期、4 人团队、约 19 万 CNY 投入**下，可交付一套满足 12 项 KPI 的离线智能检测防治系统，且投资回收期短（≈ 2 个月）、社会与生态效益显著。

**建议立项实施**。

### 12.2 实施建议

1. **严格按九阶段流程推进**，每阶段产出对应规范文档中的交付物（数据集 / 模型 / 客户端 / 文档 / 操作手册）。
2. **数据质量优先**：阶段一标注准确率 100% 是 mAP 达标的关键，农业顾问必须全程参与。
3. **小目标专项攻坚**：阶段二必须落实 P2 head + CBAM + 多尺度训练，否则红蜘蛛 / 木虱难以达标。
4. **CPU INT8 路线坚定不动摇**：FP32 ONNX 在 i3 上时延会超标，INT8 是 KPI-04 / KPI-05 的核心保障。
5. **适老化交互不打折**：3 步流程 + 大字体是项目"接地气"的灵魂，不要因为追求功能丰富而妥协。
6. **完全离线不妥协**：任何遥测 / 联网刷新需求均通过 U 盘 / 邮件离线方式实现，保住"零网络依赖"的核心卖点。
7. **法律合规前置**：若产业园未来计划对外有偿分发，须提前 1 个月启动 Ultralytics + PyQt5 商业许可采购或栈替换流程。
8. **文档与代码同步**：每个阶段变更先改文档，再改代码。
